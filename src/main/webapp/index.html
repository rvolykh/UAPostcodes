Сервіс

REST (скор. англ. Representational State Transfer, «передача репрезентативного стану») — підхід до архітектури мережевих протоколів, які забезпечують доступ до інформаційних ресурсів. Був описаний і популяризований у 2000 році Роєм Філдінгом (Roy Fielding), одним із творців протоколу HTTP. Найвідомішою системою, побудованою переважно за архітектурою REST, є сучасна Всесвітня павутина.

Документація

Description: You can search 16 day weather forecast with daily average parameters by city name. All weather data can be obtained in JSON, XML or HTML format.

API call:

api.openweathermap.org/data/2.5/forecast/daily?q={city name},{country code}&cnt={cnt}

Parameters:

INPUT:

q - city name and country code divided by comma, use ISO 3166 country codes cnt number of days returned (from 1 to 16)
z - city name and country code divided by comma, use ISO 3166 country codes cnt number of days returned (from 1 to 16)

OUTPUT:

q - city name and country code divided by comma, use ISO 3166 country codes cnt number of days returned (from 1 to 16)
z - city name and country code divided by comma, use ISO 3166 country codes cnt number of days returned (from 1 to 16)
Example
Call 7 days forecast by city name in XML format and metric units
api.openweathermap.org/data/2.5/forecast/daily?q=London&mode=xml&units=metric&cnt=7
